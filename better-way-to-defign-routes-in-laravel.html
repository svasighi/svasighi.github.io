شیوه ای بهتر برای تعریف route ها در لاراول

شادمهر وثیقی

در حالت عادی میتوانیم به شکل زیر یک route را تعریف کنیم.

Route::get('my-route', 'MyController@index');

زمان بسیار زیادیست که این شیوه توانسته route های ما را مدیریت کند. ولی در اپلیکیشن های جدیدتر لاراول این امکان وجود دارد که به شیوه ای جدید و کارآمد این تعریف را با کمک یک tuple انجام دهید.(اطلاعات بیشتر)

Route::get('my-route', [\App\Http\Controllers\SomeController::class, 'index']);

همچنین میتوان با یکبار import کردن namespace استفاده از این شیوه را ساده تر و سریع تر کرد.

use App\Http\Controllers\SomeController;

Route::get('my-route', [SomeController::class, 'index']);

در اپلیکیشن هایی که در دنیای واقعی توسعه میدهیم احتمالا از use های زیادی استفاده خواهیم کرد که ممکن است برنامه را شلوغ و مدیریت آن را سخت تر کند. البته با داشتن یک ide خوب این مشکل اجتناب پذیریست.

یادتون نره که برای استفاده از این شیوه باید RouteServiceProvider  را از لیست namespace هایی که use کردیم خارج کنیم تا لاراول به صورت دلخواه پسوند \App\Http\Controllers را به کنترلر های ما نچسباند

مزایای استفاده از این شیوه:

  1. اگر از ide پیشرفته ای مثل phpstorm استفاده کنید بایک کلیک ساده بر روی اسم کنترلر موجود درون tuple وارد کنترلر شوید
  2. زمانی که اسم controller را refactor کنید دیگر نیاز نیست این تغییرات را در route ها نیز اعمال کنید.

البته در redirect کردن ها نیز میتوان از این tuple با کمک متد action استفاده کرد. 

namespace App\Http\Controllers;

class SomeController
{
    public function someMethod()
    {
        // do some funny stuff ...

        return redirect->action([MyOtherController::class, 'index']);
    }
}

اگر از کنترلر های تک متده یا invokeable (اگر معادل درستی براش میدونید تو کامنت ها بگید) استفاده میکنید میتونید route را به شکل زیر تعریف کنید.

use App\Http\Controllers\SomeController;

Route::get('my-route', SomeController::class);

تا یادم نرفته بگم که به دلایلی - مثل کار نکردن با ide پیشرفته- ممکنه  خیلی این شیوه بدرد ما نخورد؛ ولی حتما نیازه با اون آشنا باشیم چون در بسیاری از library ها و پکیج هایی که استفاده میکنیم از این شیوه استفاده میشه ، بد نیست باهاش آشنا باشیم 😃.
